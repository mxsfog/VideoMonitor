# Таксономия контента

Таксономия нужна, чтобы признаки из OCR, транскрипта, YOLO и VLM приводились к
одним и тем же подклассам REST-контракта.

## Возрастные рамки

В проекте используются рамки, близкие к логике 436-ФЗ:

| Рамка | Смысл для проекта |
|---|---|
| `0+` | существенные риски не выявлены |
| `6+` | мягкие или контекстные упоминания без подробностей |
| `12+` | отдельные ненатуралистичные упоминания риска |
| `16+` | более тяжёлые темы без натуралистичной демонстрации |
| `18+` | признаки, которые нельзя считать детским контентом |

Категории `3+` в 436-ФЗ нет. В коде она оставлена только как совместимый alias
к `0+`, чтобы не ломать старые профили.

## Контрактные классы

REST API использует следующие верхнеуровневые классы:

- `DRUGS`;
- `DEVIANT`;
- `TERRORISM`;
- `SEX`;
- `ANTITRADITIONAL`;
- `ANTIPATRIOTIC`;
- `LUDOMANIA`.

Подклассы и словари признаков описаны в:

```text
src/content_taxonomy.py
```

## Как используется

`AGE_RATING_RULES` описывает возрастные рамки.

`RISK_RULES` связывает практические признаки с подклассами контракта:

- слова и фразы из OCR;
- слова и фразы из транскрипта;
- классы объектов YOLO;
- объекты и описания VLM.

`contract.py` не держит отдельный ручной словарь. Он вызывает функции
`keyword_matches()` и `label_to_subclass()`, чтобы вся классификационная логика
оставалась в одном месте.

## Важное ограничение

Эта таксономия не является юридическим заключением. Она нужна для учебного
pipeline и для приведения результатов к стабильному API-формату. Финальное
решение по спорному контенту требует ручной проверки.

# Курсовая работа

Тема: выявление признаков деструктивного контента в видео.

Курсовая объединяет практические задания в один прикладной сценарий: видео
загружается в систему, разбивается на кадры, анализируется по тексту, речи и
визуальным признакам, после чего формируется итоговый отчёт.

## Постановка

На вход подаётся видеофайл или ссылка на видео. На выходе система должна
вернуть:

- общий вердикт;
- временные интервалы с обнаруженными признаками;
- статистику по классам нарушений;
- отчёт в текстовом виде;
- JSON, совместимый с REST-контрактом.

Вердикты:

| Вердикт | Значение |
|---|---|
| `safe` | существенных признаков не найдено |
| `suspicious` | найдены отдельные признаки, требуется ручная проверка |
| `destructive` | найдено несколько значимых признаков |

## Pipeline

```text
видео
  │
  ├─ ПЗ 2: нарезка на кадры
  ├─ ПЗ 3: OCR титров
  ├─ ПЗ 4: распознавание речи
  ├─ ПЗ 5: детекция объектов
  ├─ ПЗ 6: классификация сцен
  ├─ ПЗ 7: LLM/VLM-классификация
  └─ ПЗ 8: постобработка
        │
        └─ агрегатор курсовой
```

Главный файл:

```text
src/coursework_pipeline.py
```

Он отвечает за запуск шагов, чтение промежуточных артефактов, расчёт признаков
и сохранение итоговых файлов.

## Запуск

Минимальный запуск:

```bash
python src/coursework_pipeline.py data/videos/example.mp4
```

Запуск с явными параметрами:

```bash
python src/coursework_pipeline.py data/videos/example.mp4 \
  --fps 1 \
  --whisper-model tiny \
  --lang ru \
  --llm-backend none
```

Запуск с VLM:

```bash
python src/coursework_pipeline.py data/videos/example.mp4 \
  --fps 1 \
  --whisper-model tiny \
  --lang ru \
  --llm-backend vlm \
  --llm-model google/gemini-2.5-flash \
  --vlm-every-n 10
```

Повторная сборка отчёта по уже рассчитанным артефактам:

```bash
python src/coursework_pipeline.py data/videos/example.mp4 \
  --skip pz2,pz3,pz4,pz5,pz6,pz7,pz8
```

## Поставка

Код на GitHub публикуется без локальных данных, моделей, runtime-выходов и
секретов. Развёрнутое решение на ВМ запускается через Docker Compose и
предоставляет Swagger UI, REST API и доступ к итоговым артефактам задач.

## Параметры

| Параметр | Назначение |
|---|---|
| `source` | путь к видео или URL |
| `--fps` | частота нарезки кадров |
| `--whisper-model` | размер модели Whisper |
| `--lang` | язык распознавания речи |
| `--llm-backend` | `none`, `ollama`, `openrouter` или `vlm` |
| `--llm-model` | имя модели для LLM/VLM |
| `--vlm-every-n` | анализировать каждый N-й кадр через VLM |
| `--skip` | пропустить уже рассчитанные шаги |
| `--job-request` | запрос REST API для фильтрации классов |
| `--job-state` | путь для сохранения состояния задачи API |

## Скоринг

Итоговый балл `score` ограничен диапазоном от `0` до `1`.

| Источник | Что учитывается | Вклад |
|---|---|---|
| ПЗ 3 | стоп-слова в титрах | до `0.3` |
| ПЗ 4 | стоп-слова в транскрипте | до `0.3` |
| ПЗ 5 | подозрительные объекты YOLO | до `0.3` |
| ПЗ 7 | доля токсичных или экстремистских LLM-меток | до `0.4` |
| ПЗ 7 | доля кадров с деструктивными VLM-объектами | до `0.4` |

Пороги:

```text
score >= 0.5  -> destructive
score >= 0.2  -> suspicious
score <  0.2  -> safe
```

Словари и соответствия классов вынесены в:

- `src/content_taxonomy.py`;
- `src/contract.py`;
- `src/coursework_pipeline.py`.

## Итоговые файлы

Каталог результата:

```text
output/coursework/<video>/
```

Состав:

| Файл | Назначение |
|---|---|
| `report.md` | краткий текстовый отчёт |
| `findings.json` | внутренние признаки скоринга |
| `job_result.json` | контрактный результат `JobResult` |
| `time_based_report.json` | временная разметка интервалов |

## Формат отчёта

`report.md` содержит:

- название видео;
- вердикт и score;
- список найденных признаков;
- статистику VLM;
- топ классов YOLO;
- ссылки на артефакты ПЗ.

`job_result.json` содержит:

- длительность обработки;
- сведения об исходном видео;
- общее число детекций;
- статистику по классам;
- список интервалов с `startSeconds`, `endSeconds`, `class`, `subclass`,
  `confidence`, `modality`.

`time_based_report.json` содержит отчёт типа `TIME_BASED_REPORT` и нужен для
показа результата по времени:

- `start_time`;
- `end_time`;
- `time_interval`;
- `subclass`;
- `confidence`;
- `type`.

## Проверка результата

Для защиты достаточно показать один завершённый прогон со статусом `DONE`.
Важно открыть не исходный файл, а итоговые артефакты:

- `report.md`;
- `job_result.json`;
- `time_based_report.json`;
- карточку задачи в Swagger UI или ответ `/api/jobs/{jobId}`.

В публичный репозиторий эти артефакты не добавляются. Они создаются локально или
на сервере после запуска pipeline.

## Ограничения

- результат не является юридической экспертизой;
- VLM зависит от качества кадров и ответа внешней модели;
- OCR может ошибаться на мелком или размытом тексте;
- YOLO использует общую модель и не закрывает специальные классы без
  дообучения;
- score нужен как инженерный индикатор для учебной задачи, а не как финальное
  модерационное решение.

## Что показывать

1. `docs/pz1.md` ... `docs/pz8.md` — связь с практическими заданиями.
2. `src/coursework_pipeline.py` — сборка шагов в один сценарий.
3. `src/contract.py` — приведение результата к контракту.
4. Swagger UI `/docs` — REST API и OpenAPI-схема.
5. `report.md`, `job_result.json`, `time_based_report.json` — итог локального
   или серверного запуска.

# Руководство для защиты

Документ задаёт порядок демонстрации проекта. Цель защиты - показать, что
практические задания объединены в воспроизводимую систему видеоанализа с REST
API, Docker-развёртыванием, контрактными моделями и проверками.

## Что открыть первым

1. `README.md`.
2. `docs/coursework.md`.
3. `docs/service_overview.md`.
4. `docs/pz1.md` ... `docs/pz8.md`.
5. Swagger UI развёрнутого сервиса: `/docs`.

Начинать лучше не с кода, а с общей схемы pipeline: видео проходит через
нарезку кадров, OCR, распознавание речи, YOLO, ResNet, LLM/VLM-классификацию и
постобработку. После этого агрегатор формирует отчёт и JSON по контракту.

## Что показывать по ПЗ

| ПЗ | Что показать | Файл |
|---|---|---|
| ПЗ 1 | обработка изображений счётчиков, варианты OpenCV, OCR | `docs/pz1.md`, `src/pz1_counters.py` |
| ПЗ 2 | получение кадров из видео с заданной частотой | `docs/pz2.md`, `src/pz2_slicer.py` |
| ПЗ 3 | OCR титров и дедупликация похожих строк | `docs/pz3.md`, `src/pz3_subtitles.py` |
| ПЗ 4 | извлечение аудио и транскрипт речи | `docs/pz4.md`, `src/pz4_whisper.py` |
| ПЗ 5 | детекция объектов YOLOv8 | `docs/pz5.md`, `src/pz5_yolo.py` |
| ПЗ 6 | классификация кадров ResNet50 | `docs/pz6.md`, `src/pz6_resnet.py` |
| ПЗ 7 | текстовая LLM и визуальная VLM-классификация | `docs/pz7.md`, `src/pz7_llm.py`, `src/pz7_vlm_gemini.py` |

Главная мысль: каждое ПЗ работает отдельно, но в курсовой они собраны в один
pipeline.

## Что показывать по курсовой

1. `src/coursework_pipeline.py` - последовательный запуск этапов.
2. `src/content_taxonomy.py` - правила классификации признаков.
3. `src/contract.py` - сборка `JobResult` и `TIME_BASED_REPORT`.
4. `src/api.py` - REST API и управление задачами.
5. `docs/contract.md` - соответствие OpenAPI-контракту.
6. `Dockerfile` и `docker-compose.yml` - развёртывание.
7. `.github/workflows/ci.yml` - автоматические проверки.

После этого нужно показать один завершённый запуск:

- `GET /health`;
- Swagger UI `/docs`;
- `POST /api/jobs`;
- `GET /api/jobs/{jobId}`;
- `GET /jobs/{jobId}/report`;
- `GET /jobs/{jobId}/artifacts/time_based_report.json`.

## Что не открывать на защите

- `.env` и любые токены.
- Полные папки `data/`, `models/`, `output/`.
- Сырые транскрипты длинных видео.
- Большие JSONL-файлы с покадровыми детекциями.
- Внутренние абсолютные пути локальной машины.

Если требуется показать результат, открывать только итоговые файлы конкретной
задачи: `report.md`, `job_result.json`, `time_based_report.json`.

## Как отвечать на вопросы

**Почему данные не лежат в GitHub?**

В репозиторий вынесены код, документация, тесты и контракт. Исходные данные,
модели и результаты выполнения могут быть тяжёлыми или приватными, поэтому они
создаются локально либо на сервере и исключены через `.gitignore`.

**Почему score не является юридическим решением?**

Score используется как инженерный индикатор для учебной системы. Финальное
решение требует ручной проверки, потому что OCR, YOLO и VLM могут ошибаться.

**Почему есть несколько моделей?**

Задача мультимодальная: текст, речь и изображение дают разные признаки. Поэтому
pipeline использует OCR, Whisper, YOLO, ResNet и LLM/VLM как независимые этапы,
а затем агрегирует результаты.

**Почему API нужен для курсовой?**

API показывает, что решение оформлено как сервис: можно создать задачу,
получить статус, забрать отчёт и удалить задачу. Это лучше, чем один локальный
скрипт без внешнего интерфейса.

**Что проверено тестами?**

Проверяются модели REST-контракта, OpenAPI-схема, сборка отчёта, безопасность
публикуемого архива, обработка задач и парсинг ответов VLM.

## Короткий порядок выступления

1. Тема и цель: система выявляет признаки деструктивного контента в видео.
2. Архитектура: ПЗ 1-8 и общий pipeline.
3. Код: показать `src/coursework_pipeline.py`, затем 2-3 ключевых этапа.
4. API: открыть Swagger UI и показать маршруты.
5. Результат: открыть завершённую задачу и итоговые JSON/Markdown-артефакты.
6. Надёжность: показать тесты, CI, Docker и исключение данных из Git.
7. Ограничения: подчеркнуть, что система помогает первичной проверке, но не
   заменяет экспертную оценку.
